Context

Dynamo in Migration from a Desktop Tool to Support Autodesk Cloud

Dynamo is an open-source visual pragramming tool in Autodesk's AEC (Architecture, Engineering & Construction) software collection. Dynamo supports data-related automation on Desktop-based 3D modeling software --Revit, Civil3D, Alias, and FormIt.

As Autodesk pivoted on its cloud platform strategic reorganization in 2021, the Dynamo team collaborate with Autodesk Platform Service to move ahead to a platform-based service. Dynamo is heading towards an exciting stage by the new opportunities brought by the cloud to push the efficiency frontier forward through batch-data selection for automation and its intuitive user experience . The 12-week project focus on researching, designing, and prototyping new Dynamo Nodes for data selection when connecting to Autodesk Construction Cloud.

Challenge

Enabling Efficient Data Selection in a Cloud-Connected Environment

As Dynamo evolved from a desktop tool to a cloud-integrated service, users struggled with managing large datasets and multi-file selections, leading to inefficiencies and a disjointed experience. They used to import data from the hero product like Revit and Civil3D Dynamo connected to. This traditional data selection methods were not designed for the scale and complexity of cloud environments, resulting in inefficiencies, limited seamlessness, and a disrupted user experience. There was a clear need for a solution that could streamline data selection processes while maintaining Dynamo’s core strengths in automation and intuitive design.

Decision -- Large Model Viewer as a piority

The users' workflow in geometry leads to a favor in Large Model Viewer as part of the Dynamo Node Collections.

Decision

After investigating various modes of data selection, the Product Manager and the team decided to prioritize the Large Model Viewer (LMV) interface. With the implementation of LMV in Dynamo, a similar workflow can easily bring the web-interface of GraphQL data selection into the Dynamo interface. This is especially useful while Dynamo transitions from a desktop service to a platform service.

1. Familiar UI 2. Visual Cue 3. Low Dev Cost

Large Model Viewer is a web-based product that many users are already familiar with, resulting in lower onboarding costs and increased user comfort.

Targeting Dynamo users within the AEC industry, a 3D geometric viewer proves to be the most effective way to convey information.

The development cost is minimized by leveraging ready-made APIs, allowing us to seamlessly transition research projects into applications with established monthly active users.

Implementation

Working with engineers, we tackled the first challenge with the authentication flow. To ensure a better user experience, we decided that users should only need to authenticate when necessary, rather than for every cloud access. Through discussions with the Data Exchange team, we learned about their implementation methods and integrated them into the Large Model Viewer (LMV) node.

We also explored different toolbars in the LMV web version to better facilitate our use cases. We incorporated the control bar for enabling 3D navigation, and included 3D geometry selection through clicking and list selection in the panel.

Additionally, we aimed to add functionality to filter geometry by properties, such as “select all the doors” or “select all the doors in Room 1.” These additions align with the workflow of Dynamo users and enhance data selection functions as Dynamo transitions from a desktop service to a platform service connected to the Autodesk Construction Cloud.
